# Проект «noLXP»

## Описание

Проект представляет собой срез части функционала lxp-платформы.

**Со стороны студента** это следующие клиентские страницы:

- страница с перечнем дисциплин и содержимым каждой из них,
- страница с детальной информацией по дисциплине, перечнем учебных тем и контрольных точек,
- страница учебной темы с обучающей информацией,
- страница контрольной точки с заданием, блоком отправки ответа и результатами,
- страница логина;

**Со стороны преподавателя** это следующий функционал в панели администрирования:

- логин в панель администрирования,
- просмотр и редактирование содержания курса (учебных тем),
- просмотр и редактирование контрольных точек курса,
- просмотр ответов на контрольные точки и выставление оценок и статусов

**Со стороны менеджера учебного процесса** это следующий функционал в панели администрирования:

- логин в панель администрирования,
- администрирование аккаунтов студентов и преподавателей,
- администрирование содержания дисциплины (учебных тем),
- администрирование контрольных точек дисциплины,
- прикрепление (и открепление) преподавателей к курсам,
- просмотр курсов, учебных тем и контрольных точек, 
- просмотр ответов студентов со статусом и оценкой преподавателя,
- выгрузка (экспорт) в табличный вид.

## Требования и критерии

### Часть 1. Моделирование предметной области (5 баллов)

#### Студенты и учебные группы (`students/models.py`)

*до 2 баллов*

Модель учебной группы `Group` должна содержать:
- название (обязательное непустое строковое поле, длина <= 15);
- курс (обязательное непустое строковое поле), допустимые значения предзаданы (через `choices`);

Модель студента `Student` должна содержать:
- отдельные поля для имени (обязательное непустое строковое поле, длина <= 20), фамилии (обязательное непустое строковое поле, длина <= 40) и отчества (необязательное строковое поле, длина <= 40);
- поле `account`, отношение 1-to-1 с аккаунтом (моделью `User`), настроено каскадное удаление;
- `group`, отношение m-to-1 с учебной группой (моделью `Group`) через внешний ключ, при удалении связанной группы должно выставляться значение `NULL`ж

Метаданные:
- поля моделей переведены через `verbose_name`;
- группы сортируются по возрастанию номера курса;
- студенты сортируются по фамилии и имени;
- добавлены индексы по курсам;

#### Дисциплины и курсы (`students/models.py`)

*до 2 баллов*

Модель `Discipline` должна содержать:
- поле `title` с названием дисциплины (обязательное непустое строковое поле, длина <= 50);
- поле `duration` с длительностью дисциплины (обязательное целочисленное поле, значение >= 0); 
- поле `curriculum` для ссылки на учебный план дисциплины (необязательное файловое поле);
- поле `created_at` с датой и временем создания (авто-добавление);
- поле `updated_at` с датой и временем последнего обновления (авто-обновление);

Модель `Course` должна содержать:
- поле `about` с описанием курса (необязательное строковое поле, длина <= 200);
- поле `course_start` с датой начала курса (необязательное поле);
- поле `updated_at` с датой завершения курса (необязательное поле);
- отношение m-to-1 с дисциплиной (моделью `Discipline`) через внешний ключ, при удалении связанной группы должно выставляться значение `NULL`;
- отношение m-to-1 с учебной группой (моделью `Group`) через внешний ключ, при удалении связанной группы использовать каскадное удаление;
- отношение m-to-1 с преподавателем (моделью `User`, `Staff` или `Teacher`) через внешний ключ, при удалении преподавателя должно выставляться значение `NULL`

Метаданные:
- поля моделей переведены через `verbose_name`;
- дисциплины сортируются по датавремени последнего обновления, далее по датевремени создания;
- курсы сортируются по кодам связанных дисциплин;
- добавлены индексы на те поля, где это способно улучшить производительность;

#### Учебные темы и контрольные точки (`content/models.py`)

*до 1 балла*

Модель `Topic` должна содержать:
- поле `content` (обязательное непустое строковое поле без ограничений);
- поле `type` (обязательное непустое строковое поле), допустимые значения (учебный материал либо контрольная точка) предзаданы через `choices`;
- поле `course`, отношение m-to-1 с курсом (моделью `Course`), при удалении связанного поля выставляется в `NULL`.

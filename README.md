# Проект «noLXP» :>

## Первые шаги

1. Создайте копию `.env.example` с именем `.env`. Задайте `PYTHON_ENVIRONMENT` равным `DEVELOPMENT`. Задайте `SECRET_KEY`, сгенерировав его через `openssl rand -hex 32` либо альтернативы.
2. Создайте и активируйте виртуальное окружение
3. Установите зависимости `pip install -r requirements.txt`
4. Примените миграции пользовательской модели `python manage.py migrate accounts`
5. Примените остальные миграции `python manage.py migrate`
6. Запустите сервер `python manage.py runserver`
7. Успехов!

## Описание

Проект представляет собой срез части функционала lxp-платформы.

**Со стороны студента** это следующие клиентские страницы:

- страница с перечнем курсов и содержимым каждого из них,
- страница с детальной информацией по курсу, перечнем учебных тем и контрольных точек,
- страница учебной темы с обучающей информацией,
- страница контрольной точки с заданием, блоком отправки ответа и результатами,
- страница логина;

**Со стороны преподавателя** это следующий функционал в панели администрирования:

- логин в панель администрирования,
- просмотр содержания курса (учебных тем),
- просмотр контрольных точек курса,
- просмотр ответов на контрольные точки и выставление оценок;

**Со стороны менеджера учебного процесса** это следующий функционал в панели администрирования:

- логин в панель администрирования,
- администрирование аккаунтов студентов и преподавателей,
- администрирование содержания дисциплины (учебных тем),
- администрирование контрольных точек дисциплины,
- прикрепление (и открепление) преподавателей к курсам,
- просмотр курсов, учебных тем и контрольных точек,
- просмотр ответов студентов с оценкой преподавателя,
- выгрузка (экспорт) в табличный вид.
